Attendance, Alerts, Events & Resources for ISB Students

A mobile-first web app built for 1000+ ISB students across 4 cohorts

What is CAPD?

A modern, mobile-friendly platform that brings together:

  • Attendance Tracking — rotating 6-digit codes, rotating every 45 seconds
  • Event Management — students see published events with full descriptions
  • Push Notifications — instant alerts for announcements and event updates
  • Cohort Isolation — PGP Hyderabad, PGP Mohali, PGP YL Y1, PGP YL Y2 keep their own data private
  • Resource Sharing — centralized hub for documents, links, and materials (coming later)

Core Features

🔐 Secure Login

Institutional email + 6-digit PIN. Students create their own PIN on first login; admins can reset it anytime.

✓ Attendance

Rotating codes that change every 45 seconds. Admin opens a session, students enter the code to check in.

📅 Events

Create events with full descriptions. Link attendance to events or publish standalone. Target all students, a cohort, or a custom list.

🔔 Notifications

Send push notifications to all students, a cohort, or a custom audience. Works on web and mobile PWA.

👥 Cohort Privacy

Each student only sees events, attendance, and notifications meant for their cohort.

📱 PWA Support

Install on any device (iOS, Android, desktop). Works offline, receives push notifications.

How Attendance Works

A TOTP-style rotating code system:

  • 6-digit code rotates every 45 seconds (configurable per session)
  • Server-side secret — never sent to the client
  • Impossible to predict — even if you have one code, you can't guess the next
  • Grace window — accepts the current code + one code back (for network latency)

Cohort & Audience Isolation

Built-in privacy via Row-Level Security (RLS):

  • Cohorts: PGP Hyderabad, PGP Mohali, PGP YL Y1, PGP YL Y2
  • Cohort-only events: A PGP Hyderabad event is invisible to Mohali students
  • Custom rosters: Upload an Excel file of emails for a club meeting — only those students see it
  • Database-level enforcement: Not just hiding UI; the database refuses to show data to students outside the audience

Push Notifications

Send instant alerts using OneSignal Web Push:

  • Audience targeting: All students, a cohort, or a custom list
  • Scheduling: Send now or at a scheduled time (once daily via cron)
  • Deep linking: Notification tap takes students to the Events page (or a specific event)
  • Works offline: PWA background sync ensures no missed alerts

Security & Privacy

  • No plaintext passwords: Supabase Auth hashes and salts everything
  • Session cookies: httpOnly, secure (HTTPS only), with a 7-day expiry
  • Attendance secrets: Stored in database, never exposed to client
  • Bulk operations (import): Only admin-level Supabase service key can create users
  • RLS policies: Database enforces audience membership at query time
  • No hardcoded secrets: All keys in environment variables, never committed to git
